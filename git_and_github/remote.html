On GitHub

- Create new repository on your GitHub account
- Checkout HTTPS adress of the repository
/* https://github.com/github_username/repository_name.git */

- Next Steps in local Command Line Interface
- - clone remote to your local
- or
- - push your local to remote

Note: Clone function does automatically remote repository setups.
Connect your local repository to remote repository

- setup link to remote repository
$ git remote add origin HTTPS-adress

- view remote repository info
$ git remote -v
Push and Pull code

- Push your code to remote
$ git push origin branch_name

- check new commits in remote
$ git fetch

- update local history to match remote
$ git merge origin/master
Clone existing remote repository to local computer

- Clone repository
$ git clone HTTPS-adress

- Clone repository and give a new name
$ git clone HTTPS-adress clone_name
The GitHub flow

- Create branch
- Commit changes
- Open pull request
- Review pull request
- Merge (and delete branch)
Remote workflow

1. Fretch and merge changes from remote repository
2. Create a branch to work on a new project feature
3. Develop the feature on your branch and commit your work
4. Fetch and merge from the remote repository again (in case new commits were made while you were working)
5. Push your branch up to the remote for review
