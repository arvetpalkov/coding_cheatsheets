Function Declaration 


  function myFunction() {
    console.log('Hello!');
  }
- function keyword
- identifier, name of the function
- parentheses for parameters, ()
- function body inside curly brackets, {}

Call function 

  myFunction();
Parameters and Arguments

Declare function with paramaters: 

  function calculateArea(width, height) {
    console.log(width * height);
  }
Call function with arguments: 

  calculateArea(8, 5);
Variables as argumens: 

  const rectWidth = 8;
  const rectHeight = 5;

  calculateArea(rectWidth, rectHeight);
Default Parameters 

  function greeting (name = 'stranger') {
    console.log(`Hello, ${name}!`)
  }
  
  greeting('Nick') // Output: Hello, Nick!
  greeting() // Output: Hello, stranger!
Return 

  function calculateArea(width, height) {
    const area = width * height;
    return area;
  }
By default function return value is undefined.

There can be multiple return scenarious: 

  function rectangleArea(width, height) {
    if (width < 0 || height < 0) {
      return 'You need positive integers to calculate area!';
    }
    return width * height;
  }
Function expression 
  
  const calculateArea = function(width, height) {
    const area = width * height;
    return area;
  }
Arrow Function 

  const rectangleArea = (width, height) => {
    let area = width * height;
    return area;
  };
Concise Body Arrow Functions

A single parameter do not need parentheses 
  
  const functionName = () => {};

  const functionName = param1 => {};

  const functionName = (param1, param2) => {};
A single line function body do not need curly brackets and return keyword.
The function: 

  const squareNum = (num) => {
    return num * num;
  };
can be writen : 

  const squareNum = num => num * num;
Blocks and Scope

A block is the code found inside a set of curly braces {}.

Global Scope / Block Scope

Global variables are declared outside of blocks. They can be accessed fron any where in the program.

Local variables are declared inside a block. They are only available to the code inside of the same block.

Scope Pollution

Too many variables in global namespace can cause problem to cotrol variables in different scopes.
To prevent mistakes, try to avoid global variables if it is possible.
