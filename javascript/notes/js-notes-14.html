Debuging JavaScript code

Three of seven most common built-in error types:

- SyntaxError: Some typo in the code.
- ReferenceError: Fault with variable or function declaration.
- TypeError: If you attempt to perform an operation on a value of the wrong type.

More info: Mozilla docs built-in Errors

Silent Bugs

Silent bugs don't throw errors but code doesn't work right.

One of techniques to locate silent bugs is to add console.log() statements to the code to inspect program flow and content of variables.

When you are fixing bugs, you may find some answers in the JavaScript documentation: MDN JavaScript Documentation

Another good way to find solutions is to Google things.

Another good site to search information is Stack Owerflow.

Error Handling

In JavaScript, there are built-in error objects that have a name and message property.

Creating or logging error will not stop running the program. 

  console.log(Error('Your password is too weak.'));
  // Prints: Error: Your password is too weak.
or 

  console.log(new Error('Your password is too weak.'));
  // Prints: Error: Your password is too weak.
The throw Keyword

A thrown error will cause the program to stop running. 

  throw Error('Somethig wrong happend');
  // Error: Something wrong happend
The try...catch Statement

In JavaScript, we use try...catch statement to anticipate and handle errors. 

  try {
    throw Error('This error will get caught');
  } catch (e) {
    console.log(e);
  }
  // Prints: Error: This error will get cought

  console.log('Program runs after error handling');
With this structure we can handle errors and keep program running.

We can also use a try...catch statement to handle built-in errors that are thrown by the JavaScript engine. 

  const someVar = 'Cannot be reassigned';

  try {
    someVar = 'Still going to try';
  } catch(e) {
    console.log(e);
  }
  // Prints: TypeError: Assignment to constant varible.
